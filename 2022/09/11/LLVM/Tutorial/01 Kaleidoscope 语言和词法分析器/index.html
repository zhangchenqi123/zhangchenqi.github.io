    1.1 Kaleidoscope语言

    Kaleidoscope（万花筒）语言是一种程序语言，它允许你定义函数，使用条件判断，使用数学公式等。在整个教程中，我们将扩展 Kaleidoscope 以支持 if/then/else 构造、for 循环、用户定义的运算符、带有简单命令行界面的 JIT 编译、调试信息等。

    为了保持简洁，Kaleidoscope语言只有64位浮点数这一种数据类型，因此，所有值都是隐式双精度的，并且该语言不需要类型声明。这为该语言提供了非常简单的语法。例如，以下简单示例计算斐波那契数：

    # 计算斐波那契数列
def fib(x)
  if x < 3 then
    1
  else
    fib(x-1)+fib(x-2)

# 计算第40个数
fib(40)

    我们还允许 Kaleidoscope 调用标准库函数——LLVM JIT 使这变得非常容易。这意味着您可以在使用函数之前使用“extern”关键字来定义函数（这对于相互递归的函数也很有用）。例如：

    extern sin(arg);
extern cos(arg);
extern atan2(arg1 arg2);

atan2(sin(.4), cos(42))

    语法通过如下的推导规则来定义：非终结符号（nonterminal- symbol）在左边（Left Hand Side——LHS），终结符号（terminal- symbol）和非终结符的组合在右边（Right Hand Side——RHS）；当遇到一个LHS，就会根据推导规则生成与之对应的RHS。

    1. 一个算术表达式可以是一个数字常量：

    numeric_expr := number

    1. 括号表达式会在左右括号之间包含一个表达式：

    paran_expr := '(' expression ')'

    1. 一个标识符（identifier）表达式会是一个标识符或者函数调用：

    identifier_expr

    := identifier

    := identifier '('expr_list ')'

    1. 如果标识符_expr是函数调用，它要么没有参数，要么有一个由逗号分隔的参数列表：

    expr_list

    := (empty)

    := expression (',' expression)*

    1. 另外还有一些基本表达式，它们可以是标识符表达式、算术表达式，或者括号表达式，语法会从基本表达式开始展开。

    primary := identifier_expr

    :=numeric_expr

    :=paran_expr

    1. 一个表达式可以是一个二元表达式：

    expression := primary binoprhs

    1. 一个RHS二元运算则是二元运算符和表达式的组合：

    binoprhs := ( binoperator primary )*

    binoperators := '+'*'-'*'*'*'*'

    1. 函数声明具有如下的语法：

    func_decl := identifier '(' identifier_list ')'

    identifier_list := (empty)

    := (identifier)*

    1. 函数定义使用def关键字，其后是函数声明和定义了函数体的表 达式：

    function_defn := 'def' func_decl expression

    1. 最后，需要一个顶层的表达式，以生成所有种类的表达式：

    toplevel_expr := expression

    1.2 Lexer - 词法分析器

    当实现一种语言时，首先要做的事就是处理文本文件并识别其内容，执行此操作的传统方法是使用词法分析器（Lexer）将输入分解为词法单元（token）。词法分析器返回的每个token都包含一个token代码和可能的一些元数据（例如数字的数值）。首先，我们定义词的可能类型（即token code的取值）：

    // 如果见到未知种类的字符，就token值它的ascii值，否则返回以下值之一
enum Token {
  // 文件结尾，token值为-1
  tok_eof = -1,

  // 关键字“def”的token值为-2
  // 关键字“extern"的token值为-3
  tok_def = -2,
  tok_extern = -3,

  // 标识符（变量、常量、函数名）的token值为-4
  // 数字类型的token值为-5
  tok_identifier = -4,
  tok_number = -5
};


static std::string IdentifierStr;
static double NumVal;

    如果我们见到了如下表达式： num = 1，则它会被分解成如下：

    [-4,num],['=', =],[-5,1]

    我们的词法分析器返回的每个标记要么是 Token 枚举值之一，要么是一个“未知”字符，如“+”(返回这个符号的ASCII码)，如果当前的token是一个标识符，则 IdentifierStr 全局变量保存标识符的名称。如果当前的token是一个数字（比如1.1），则NumVal表示它的值。我们为了简便而用全局变量，但是当要真正构建一个语言的时候，这不是一个好的选择。

    词法分析器的实际实现是一个名为 gettok() 的函数。调用 gettok() 函数从标准输入返回下一个词法单元。

    gettok() 函数每次从标准输入中读取一个完整的词法单元

    首先要跳过所有的空格，然后识别标识符”def”和“external”：

    // gettok：返回从标准输入的下一个词法单元
static int gettok() {
    static int LastChar = ' ';

    // 跳过空格
    while (isspace(LastChar))
        LastChar = getchar();

    if (isalpha(LastChar)) { // 标识符: [a-zA-Z][a-zA-Z0-9]*
        IdentifierStr = LastChar;
        while (isalnum((LastChar = getchar())))
            IdentifierStr += LastChar;

        if (IdentifierStr == "def")
            return tok_def;
        if (IdentifierStr == "extern")
            return tok_extern;
        return tok_identifier;
    }

    请注意，这段代码在对标识符进行词法分析的时候会设置全局“IdentifierStr”。

    数值的处理是类似的：

    if (isdigit(LastChar) || LastChar == '.') {   // Number: [0-9.]+
    std::string NumStr;
    do {
        NumStr += LastChar;
        LastChar = getchar();
    } while (isdigit(LastChar) || LastChar == '.');

    NumVal  = strtod(NumStr.c_str(), 0);
    return tok_number;
}

    当读入一个数值时，我们用C语言的strtod函数来把NumVal（字符串类型）转换成数值。注意，这里没有错误类型检查，如1.23.45.56将会处理成1.23，之后大家可以对这点自行扩展。

    接下来处理注释

    if (LastChar == '#') {
  // 若看到了#，则从当前位置到这行结束都是注释
  do
    LastChar = getchar();
  while (LastChar != EOF && LastChar != '\n' && LastChar != '\r');

  if (LastChar != EOF)
    return gettok();
}

    处理注释的时候，直接跳到这行的结尾，然后返回下一个token。

    如果输入不满足上面的任何一种情况，它要不是个操作符（比如“+”），或是这个文件的结尾。用如下代码处理：

    // 检查是不是文件的结尾
if (LastChar == EOF)
    return tok_eof;

  // 否则，就返回这个字符的ascii值
  int ThisChar = LastChar;
  LastChar = getchar();
  return ThisChar;
}
